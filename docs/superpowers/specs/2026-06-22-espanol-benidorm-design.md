# Español Benidorm — дизайн (спецификация)

Дата: 2026-06-22
Статус: утверждён к реализации (v1)

## 1. Цель

Личное PWA-приложение для изучения испанского языка для одного пользователя
(уровень A2–B1), живущего в Валенсийском сообществе рядом с Бенидормом. Акцент —
быстрый прогресс, удобство, ежедневная практика и **местный (валенсийский/
аликантийский) вариант разговорного испанского**.

Не многопользовательское. Без аккаунтов и логина.

## 2. Ключевые решения (зафиксированы в брейнсторме)

| Решение | Выбор |
|---|---|
| AI-движок | Claude API (грамматика, диалоги, автозаполнение, коучинг речи) |
| Хранение API-ключа | Только на телефоне (вводится один раз в Настройках) |
| Напоминания | Отложено: web-push убран из scope; пользователь настраивает через Календарь/Напоминания iOS вручную |
| Целевая платформа (сейчас) | Chrome (десктоп) — полная поддержка Web Speech API, включая распознавание речи для «Логопеда». iOS-установка позже. |
| Уровень ученика | A2–B1 |
| Технический подход | Чистый PWA без сборки, IndexedDB, Web Speech API, Service Worker |
| Оценка произношения | Разборчивость (браузерное распознавание) + AI-коучинг |

## 3. Архитектура

Чистый клиентский **PWA** из статичных файлов. Бэкенда нет.

- Устанавливается на iPhone: Safari → «Поделиться» → «На экран Домой».
- Запросы к Claude API напрямую из браузера, ключ пользователя, заголовок
  `anthropic-dangerous-direct-browser-access: true`.
- Озвучка — Web Speech API `SpeechSynthesis` (голос `es-ES`, бесплатно).
- Распознавание речи — Web Speech API `SpeechRecognition` (`es-ES`).
- Хранилище — IndexedDB (словарь, состояние SRS, история, настройки, лог ошибок).
- Service Worker — офлайн-оболочка + push-напоминания.
- Хостинг — GitHub Pages (бесплатно).

### Модель и стоимость

По умолчанию `claude-haiku-4-5` (дёшево/быстро: грамматика, автозаполнение,
коучинг), для диалогов опционально `claude-sonnet-4-6`. Модель выбирается в
Настройках. Расход — центы за активное использование.

## 4. Экраны (нижняя панель вкладок, интерфейс на русском)

5 видимых вкладок с группировкой:

1. **Учить** — SRS-очередь повторения на сегодня
2. **Словарь** — двусторонний перевод RU→ES и ES→RU + добавление слов
3. **Речь** — подвкладки: **Аудио** (диалоги на слух) и **Логопед** (фонетика/произношение)
4. **Грамматика** — проверка фраз
5. **Ещё** — Прогресс, Настройки

## 5. Модули (каждый — отдельный файл, одна задача)

- `app.js` — оболочка, hash-роутер, **реестр модулей** (см. §9 Расширяемость)
- `claude.js` — клиент API: `checkGrammar()`, `generateDialogue()`, `enrichWord()`,
  `gradeAnswer()`, `gradeSpeech()`
- `db.js` — обёртка IndexedDB + версионирование схемы (миграции)
- `srs.js` — интервальное повторение (SM-2-lite), чистые функции (юнит-тесты)
- `dictionary.js` — двусторонний перевод (RU→ES и ES→RU, переключатель
  направления + автоопределение языка ввода), автозаполнение карточки, CRUD слов
- `listening.js` — генерация диалога + TTS-воспроизведение + вопрос на понимание
- `speech.js` — **Логопед**: чтение вслух, оценка разборчивости, AI-коучинг,
  упражнения (минимальные пары, скороговорки, ударения), ораторские заметки
- `grammar.js` — проверка фразы + объяснение по-русски + лог ошибок в «слабые темы»
- `reminders.js` — ежедневное локальное уведомление по расписанию
- `progress.js` — стрик, выучено слов, очередь на сегодня, слабые темы
- `settings.js` — API-ключ, время напоминания, уровень, голос, модель
- `prompts.js` — системные промпты, включая диалект и фонетический коучинг

## 6. Потоки данных

- **Слово:** ввод RU или ES → `enrichWord` (перевод, пример, род, спряжение,
  местный вариант) → превью карточки → сохранение в IndexedDB → в SRS-очередь.
- **Повторение:** напоминание → открытие → очередь «на сегодня» → оценка ответа
  обновляет расписание SRS.
- **Аудио:** тема (выбор/авто) → Claude генерит диалог с местным колоритом →
  TTS-озвучка → транскрипт по кнопке → вопрос на понимание.
- **Грамматика:** ввод фразы → Claude правит → объяснение по-русски → ошибка
  пишется в слабые темы.
- **Логопед:** эталонная фраза → запись речи (SpeechRecognition) → сравнение с
  эталоном = оценка разборчивости (% совпадения) → `gradeSpeech` (Claude): какие
  звуки хромают, советы по ударениям/темпу/паузам/интонации + упражнение.

## 7. Диалект (Бенидорм / Валенсия)

Общий системный промпт инструктирует Claude: база — стандартный кастильский
испанский, но подсвечивать местные валенсийские/аликантийские словечки,
выражения района Бенидорма, помечать отличия; уровень A2–B1. Фонетический промпт
учитывает звуки, сложные именно для русскоязычных (раскатистое `rr`, межзубное
`c/z`, мягкие `ll/y`) и местный выговор.

## 8. Обработка ошибок

- Нет API-ключа → подсказка перейти в Настройки.
- Нет сети → словарь и SRS работают офлайн; AI-функции мягко сообщают об ошибке.
- Нет голоса `es` / нет распознавания → текстовый fallback.
- Ошибки API (лимит, неверный ключ) → понятное сообщение.
- iOS push: карточка-инструкция «добавь на экран Домой и разреши уведомления»;
  деградация до внутреннего напоминания.

## 9. Расширяемость (на будущее)

Чтобы новые блоки добавлялись легко, без переписывания ядра:

- **Реестр модулей.** `app.js` хранит массив `features`. Каждый модуль
  регистрируется объектом: `{ id, title, icon, group, render(container), order }`.
  Добавить новый блок = создать `js/<name>.js`, вызвать `registerFeature({...})`,
  подключить `<script>`. Навигация и роутинг строятся из реестра автоматически.
- **Реестр типов упражнений.** SRS и Логопед используют общий список типов
  упражнений (`exerciseTypes`), каждый со своим `render` и `grade`. Новый тип
  упражнения добавляется регистрацией, без правки очереди.
- **Версионирование схемы БД.** `db.js` хранит `DB_VERSION` и применяет миграции
  по возрастанию. Новые поля карточки / новые хранилища добавляются миграцией, не
  ломая существующие данные.
- **Промпты вынесены в `prompts.js`** — добавление нового AI-сценария = новая
  функция-промпт, ядро не трогается.
- **Экспорт/импорт данных** (JSON) закладывается в `db.js` как точка для будущего
  бэкапа/синка.

## 12. Отложено на v2 (банк идей)

статистика-графики, установка на iOS Safari (после стабилизации в Chrome).
