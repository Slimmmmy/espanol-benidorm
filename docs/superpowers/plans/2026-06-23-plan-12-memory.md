# Español Benidorm — План 12: Память наставника (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Постоянные «заметки о ученике» (имена друзей/семьи, цель, предпочтения, что даётся трудно), которые ВСЕГДА идут наставнику и учителю в контекст. Память автоматически пополняется из чата и доступна для просмотра/правки/очистки в Настройках. Хранится в `settings.tutorMemory` (синхронизируется, объединяется без потерь).

**Architecture:** Поверх Планов 1–11. Память — массив коротких строк-фактов в `settings`. `buildProfile()` начинает включать `memory`, поэтому она автоматически попадает во все AI-сценарии (чат, уроки, задания). После ответа в чате — фоновый вызов `extractMemory` обновляет заметки. Слияние памяти — union без дублей (в `merge.js`, тест).

**Tech Stack:** HTML/CSS/JS (ES-модули, без сборки), IndexedDB, Claude API, Node 20 `node --test`.

function mergeMemory(a, b) {
  const seen = new Set();
  const out = [];
  for (const n of [...(a || []), ...(b || [])]) {
    const k = String(n).trim();
    if (!k || seen.has(k.toLowerCase())) continue;
    seen.add(k.toLowerCase());
    out.push(k);
  }
  return out.slice(-50);
}

- [ ] **Step 2: Добавить фоновую функцию обновления памяти**

Перед функцией `send` добавить:
```js
async function refreshMemory(userMsg, assistantMsg) {
  try {
    const existing = await getMemory();
    const r = await extractMemory(existing, userMsg, assistantMsg);
    if (r && Array.isArray(r.notes)) {
      await saveMemory(r.notes);
      autoSync();
    }
  } catch (e) { /* тихо: память — необязательная функция */ }
}
```
